
# ADD MOUSE-CURSOR-FOLLOWER CONFIG (v0.5.0)
- Add a timeout option to hide the mouse-cursor-following translation from the Electron app.
- With a short explanation saying that the feature is available through the native app, users who are using the web app can also learn that native app exclusive features exist.





---





Fren, you are a web app master. I need your help to add a new config option.


## 1. Goal
01. Add a config option for the mouse-cursor-follower using a collapse component.
02. The option for the mouse-cursor-follower is a timeout to hide the mouse-cursor translation after translation.
03. Use a short explanation that the feature is available through the native app so that users who are using the web app can also learn that the native app unlocks extra features.



## 2. Context
01. Add a new option to the webapp's config component, using the collapse component.
	01-01. Webapp config: `src/01_webapp/app/component/config/`.
	01-02. Collapse component: `src/01_webapp/component/collapse/collapse.component.scss`.
	01-03. Add the option to the end of the config tab.
02. The new option is a timeout option for the mouse-cursor-follower.
	02-01. After the timeout, the mouse-cursor-follower window will be hidden.
	02-02. The timeout does not clear the label text by itself.
	02-03. When a new translation arrives, clear any previous hide timer and start a new timer based on the latest timeout value.
	02-04. The timeout logic stays in the webapp. Electron only receives the native hide/clear command.
03. Use a simple dropdown form for the new option:
	03-01. `Do not hide`: do not hide the label.
	03-02. `5 sec`: hide the label after 5 seconds.
	03-03. `10 sec`: hide the label after 10 seconds (default).
	03-04. `20 sec`: hide the label after 20 seconds.
	03-05. `30 sec`: hide the label after 30 seconds.
	03-06. `1 min`: hide the label after 60 seconds.
04. Set the collapse component default behavior:
	04-01. If built as a web app, then default is close for the collapse component (also, disable the dropdown form with a warning message).
	04-02. If built as a native app (Electron), then default is open for the collapse component.
	04-03. Use the helper: `src/01_webapp/common/env/env.helper.ts`.
	04-04. Add a bit more space above the collapse component because this section is a different context.
05. For the texts of the collapse component:
	05-01. Title: `Exclusive Native App Features`.
	05-02. The meta description (`.c-collapse-meta`) must be general and indicate that the settings in this section are available through the native app.
	05-03. Label: `Hide Mouse-Cursor Translation After:`
	05-04. Description: `After translation finishes, the translated text appears near your mouse cursor, so you can keep reading without looking back at the native app. This setting controls how long it stays visible.`
	05-05. Webapp warning: `Available in the native app. This web app setting is disabled because mouse-cursor translation needs the native app.`
06. To communicate between the webapp and the native app, use `native-event`.
	06-01. `src/01_webapp/common/native-event/`.
07. Add a separate explicit native hide/clear contract.
	07-01. Recommended names: `NATIVE_MOUSE_CURSOR_FOLLOWER_CLEAR_EVENT` & `NATIVE_MOUSE_CURSOR_FOLLOWER_CLEAR_IPC_CHANNEL`.
	07-02. The event name may use `CLEAR`, but the timeout behavior must hide the mouse-cursor-follower window without clearing the label text.



## 3. Rules
01. Understand the existing IPC logic.
	01-01. `NATIVE_TRANSLATION_OUTPUT_EVENT` in `src/01_webapp/common/native-event/native-event.constant.ts`.
	01-02. `NATIVE_TRANSLATION_OUTPUT_IPC_CHANNEL` in `src/02_electron/src/app/app.constant.ts`.
02. Keep the native bridge structure future-proof for manually clearing the mouse-cursor-follower label.
	02-01. Do not use the timeout number as a command signal.
	02-02. Hide/clear must be represented by a separate explicit native event / IPC command, not by an empty translation string or timeout value.
03. The Electron app only needs to handle the event `NATIVE_MOUSE_CURSOR_FOLLOWER_CLEAR_EVENT`.
	03-01. The timeout logic stays in the webapp.
	03-02. The Electron app hides the mouse-cursor-follower window when the event `NATIVE_MOUSE_CURSOR_FOLLOWER_CLEAR_EVENT` is received.
	03-03. The Electron app must not clear the label text when hiding the mouse-cursor-follower window.
04. There is no bugs in the existing app, so be careful when you make changes.
05. Respect the current coding style.
06. The best structure is easy-to-delete.
07. Since this is Linux environment (no GUI), do not build the Electron app. I will do it by myself.
