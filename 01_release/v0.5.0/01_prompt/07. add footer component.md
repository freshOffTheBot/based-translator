
# ADD FOOTER COMPONENT (v0.5.0)
- To show more info, add a footer component.





---





Fren, you are a web app master. I need your help to add a footer to the webapp.


## 1. Goal
01. Add a footer component to `src/01_webapp/common/footer/`.
02. The footer is located at the bottom of the screen (reserved bottom).
03. The footer is designed to be thin, just enough to show 3 lines of text.
04. The footer is as wide as the header (the same width).



## 2. Context
01. The footer related files:
	01-01. `src/01_webapp/common/footer/footer.component.ts`
	01-02. `src/01_webapp/common/footer/footer.html`
	01-03. `src/01_webapp/common/footer/footer.scss`
02. The footer has 3 sections:
	02-01. The left section (left aligned).
	02-02. The right section (right aligned).
	02-03. The center section.
03. The left section of the footer shows a text:
	03-01. `<span>Made with ❤️ in 🇺🇸</span>`.
04. The right section of the footer shows a list of links.
	04-01. `<a href="https://github.com/freshOffTheBot/based-translator" target="_blank"><span>[GitHub]</span></a>`.
	04-02. `<a href="https://www.youtube.com/@freshOffTheBot" target="_blank"><span>[YouTube]</span></a>`.
	04-03. The links are stacked in vertically.
05. The center section of the footer shows the app's version.
	05-01. Add a build-time `__APP_VERSION__` constant sourced from root `package.json` in both webapp and Electron renderer Vite configs, plus a declaration in `env.d.ts`.
	05-02. Add a new function to get the app version in `env.helper.ts`.
06. All the texts in the header use font size: `$font-size-500`.
	06-01. Use the font `$fontPixel`.



## 3. Rules
01. Wire the footer into the home shell:
	01-01. Add a footer mount point between after `#home-app`.
	01-02. Use `footer.component.ts` only as a static render helper with no footer state, no event listeners.
	01-03. Benchmark the existing header mount logic in `src/01_webapp/home/home.component.ts`.
02. Use CSS only for the footer (no TypeScript).
	02-01. The main CSS class name is: `.c-footer` (`.c-` as a common component).
	02-02. Additional CSS class name is: `.c-footer-*` (`.c-footer-` as prefix).
	02-03. The only logic for `footer.component.ts` is `initializeFooterComponent`, just like `src/01_webapp/common/header/header.component.ts`.
03. Unlike the header, the footer is not fixed - it just sits at the bottom.
04. Use symantic HTML tags like; `<footer>`.
05. The current webapp doesn't have bugs, so be careful when you make changes.
06. Consider the mobile screen.
