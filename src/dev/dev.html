Components > Card

Cards are intentionally minimal: one surface, one border, compact spacing, and small metadata helpers.

Default

Simple content container

Use the base card when you only need a title, body copy, and a quiet surface. 

<div class="c-card">
	<div class="c-card-body">
		<p class="c-card-eyebrow">Default</p>
		<h3 class="c-card-title">Simple content container</h3>
		<p class="c-card-text">Use the base card...</p>
	</div>
</div>

Structured

Header, body, footer

Compose sections when the card needs hierarchy, metadata, or actions.

Components > Button

Buttons use the same minimal dark surface language: one clear primary action, one quiet ghost option, and a few semantic alternatives for utility cases. 

<button class="c-button" type="button">Primary</button>
<button class="c-button c-button-ghost" type="button">Ghost</button>
<button class="c-button c-button-muted" type="button">Muted</button>
<button class="c-button c-button-danger" type="button">Danger</button>

<button class="c-button c-button-block" type="button">Block Button</button>

Usage

Pair actions with clear priority

Reserve the green button for the main action, then use muted or ghost styles for secondary flows. 

<div class="dev-actions">
	<button class="c-button" type="button">Save changes</button>
	<button class="c-button c-button-ghost" type="button">Cancel</button>
</div>

Components > Tab

Tabs use the same compact surface language as buttons and cards: quiet by default, then clearly active when a section is selected.

Active Panel

Recording stays the default workflow

Use the active tab to anchor the current context, then render the matching content in the panel below. 

<div class="c-tab">
	<div class="c-tab-list" role="tablist" aria-label="Translator views">
		<button class="c-tab-button c-tab-button-active" type="button" role="tab" aria-selected="true">Recording</button>
		<button class="c-tab-button" type="button" role="tab" aria-selected="false">Configuration</button>
	</div>

	<div class="c-tab-panel" role="tabpanel">
		<h3 class="c-card-title">Recording stays the default workflow</h3>
		<p class="c-card-text">Render the matching panel content here.</p>
	</div>
</div>
