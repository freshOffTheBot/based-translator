Minimal CSS Components

Shared card and form atoms built only with the typography token file. Use these examples as the default markup shape for future UI work.

Card Atoms

Cards are simple section shells for grouping related content with clear borders and spacing.

Card with Header, Body, and Footer

Recorder Status

Use the header for the main label and short context.

The body holds the primary content area. Keep text aligned to the shared spacing atoms.

Footer copy works well for secondary notes or actions.

<article class="c-card">
	<div class="c-card-header">
		<h3 class="c-card-title">Recorder Status</h3>
		<p class="c-card-description">Use the header for the main label and short context.</p>
	</div>
	<div class="c-card-body">
		<p class="c-card-description">The body holds the primary content area.</p>
	</div>
	<div class="c-card-footer">
		<p class="c-card-description">Footer copy works well for secondary notes or actions.</p>
	</div>
</article>

Body-Only Card

Transcription Output

Use a single body section when the card does not need framing content above or below. 

<article class="c-card">
	<div class="c-card-body">
		<h3 class="c-card-title">Transcription Output</h3>
		<p class="c-card-description">Use a single body section when framing content is not needed.</p>
	</div>
</article>

Form Atoms

Form pieces share one surface system so fields, help text, errors, and actions read as one unit.

Input Field

Help text should stay short and instructional.

<form class="c-form">
	<div class="c-form-field">
		<label class="c-form-label" for="api-key">OpenAI API Key</label>
		<input class="c-form-input" id="api-key" name="api-key" type="password" placeholder="sk-..." />
		<p class="c-form-help">Help text should stay short and instructional.</p>
	</div>
</form>

Textarea with Error

Include the {{transcription}} placeholder before submitting.

<form class="c-form">
	<div class="c-form-field">
		<label class="c-form-label" for="translation-template">Translation Template</label>
		<textarea class="c-form-textarea" id="translation-template" name="translation-template" placeholder="Translate {{transcription}} into English."></textarea>
		<p class="c-form-error">Include the {{transcription}} placeholder before submitting.</p>
	</div>
</form>

Action Row

Buttons use the same type scale and focus treatment as the fields.

<div class="c-form-actions">
	<button class="c-form-button" type="button">Start Recording</button>
	<button class="c-form-button" type="button" disabled>Transcribing Audio...</button>
</div>
